ResultsDirectorySetPath
Specify the directory to save the test results. The entire path is created, if not already present. Do not use a URI for the -ResultsDirectory. It must be a native file path. Otherwise, the sequencer will get stuck in a PAUSE state.

Argument Description
uri Path to result directory

Example:
[info homeDir uri]iTest/DNS-Benchmark
expectSubfolder 1=expect results path to contain "Results" subfolder
